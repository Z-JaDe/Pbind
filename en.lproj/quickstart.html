Prepare: Install CocoaPods

If you hadn't install Cocoapods, install it.

Step 1: Init project

In this step, you will:

  • Create Application
  • Integrate Pbind
  • Integrate PBLiveLoader

Create Application

Open Xcode, Click File->New->Project... to create a Single View Application. Set the Product Name, for this sample we set as HeroTour (you can named whatever you want).

Integrate Pbind

Close the project and go into command line, run the command to initialize a Podfile:

pod init

Modify the generated Podfile, add dependency of Pbind:

target 'HeroTour' do
  # Uncomment the next line if you're using Swift or would like to use dynamic frameworks
  # use_frameworks!

  # Pods for HeroTour
  pod 'Pbind'

end

Run pod install to install Pbind.

Integrate PBLiveLoader

PBLiveLoader is a directory watchdog, use to reload page instantly.

Firstly, install the Pbind CLI:

gem install pbind

cd to your project directory and run:

pbind watch

After this the project would apply following changes:

├── HeroTour
│   └── Info.plist (Mofify)
├── PBLocalhost (Add)
│   └── ignore.h
└── PBLiveLoader (Add)
    ├── PBDirectoryWatcher.h
    ├── PBDirectoryWatcher.m
    ├── PBLiveLoader.h
    └── PBLiveLoader.m

Thereinto:

  • PBLocalhost - Local mock server, we can add json files under it to mock API response, and the ignore.h use to configure excludes.
  • PBLiveLoader - Instant reloader, watch plist, json file changes and reload instantly.
  • Info.plist - Watch path settings, add two fields:
    • PBResourcesPath - the resources path of current project
    • PBLocalhost - the local mock server path

Step 2: Work with plist

Reopen [your-project].xcworkspace:

open HeroTour.xcworkspace

Modify the ViewController.h and make it extends with PBTableViewController:

  #import <UIKit/UIKit.h>
  #import <Pbind/Pbind.h>

  @interface ViewController : PBTableViewController


  @end

The PBTableViewController takes PBTableView as root view by what we can configure the TableViewby Plist.

Modify the ViewController.m and set the plist name:

  - (void)viewDidLoad {
      [super viewDidLoad];
      [self setPlist:@"main.tableview"];
  }

Create the main.tableview.plist, add configurations:

KeyTypeValue
RootDictionary(1 item)
propertiesDictionary(1 item)
headersArray(1 item)
Item 0Dictionary(1 item)
clazzStringUILabel
heightString100
propertiesDictionary(0 items)
textStringHello Pbind!
textColorString#fff
backgroundColorString#333
textAlignmentString:center

If you want to copy above codes, please select the Root node and click the poping [COPY] button to copy. Fold the Root node in Xcode, select it and press [⌘+v] to overwrite.

Step 3: Start application

Choose a simulator, press [⌘+R] to run.

Step 4: Make some instant changes

Cause there is no navigation bar now, the root view is overlaying the status bar. As we know the status bar is about 20px height, let's add the margin to resolve it:

KeyTypeValue
clazzStringUILabel
heightString100
marginString{20,0,0,0}

[⌘+S] to save:

Seems not working well that we just push down a bit of the root view, what's wrong? This is because Pbind takes 1080 as default base width which is about the size of @3x, so we need to set the top margin to 60 here:

KeyTypeValue
marginString{60,0,0,0}

This feature is for configurating the UI metrics more convenient, you can set the base width once by:

[Pbind setSketchWidth:the_width]; // The width of the sketch provided by your UI designer

and then directly configure all the metrics marked in the sketch. Pbind will adjust them for devices automatically.

Save again:

You can try to change the background/foreground color of the text by yourself and enjoy it!

More details about the constants configuration can refer to Constants and Row mapper.

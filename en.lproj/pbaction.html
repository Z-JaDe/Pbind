Action Component

the action component is implemented by PBAction, used to define the action standard by what we can configure the interactions.

Base names:

name type note
type String action type
disabled BOOL whether disabled, default is NO, assigned by an expression as usuall
params Dictionary parameters for each action
next.* Dictionary the next action, the * is determined by different actions, usually is done

PBNotificationAction

Notification action is used to configure the interactions of notification center. Accept types:

notify

Send a notification, extension names:

name type note
name String the notification name to send
target String the object carried by the notification
params Dictionary the userInfo carried by the notification

watch

Observe a notification, extension names:

name type note
name String the notification name to observe
target String the object carried by the notification
next.done String the next action triggered by received the specify notification

PBNavigationAction

Navigation action is used to configure the interactions of page routing. Accept types:

push

Push a new page, extension names:

name type note
target String the target route, usually is a ViewController class name
params Dictionary the properties used to initialize the target controller
next.done Dictionary the next action triggered by push done

pop

Pop page, extension names:

name type note
target String the target route, if not set, back to the last controller
params Dictionary the properties used to update the target controller
next.done Dictionary the next action triggered by pop done

present

Present a model page, the configurations are same as push.

dismiss

Dismiss the model page, the configurations are same as push except doesn't accept target.

PBAlertAction

Alert action is used to configure the interactions of UIAlertController. Accept types:

alert

Display a model dialog, related to UIAlertControllerStyleAlert, accept params:

name type note
title String the title of the dialog
message String the content of the dialog
buttons Array the button titles for the dialog

and the next param format:

name type note
next.[0-N] Dictionary the next action triggered by click the specified index

sheet

Present a model sheet, related to the style of UIAlertControllerStyleActionSheet. All the names are same as alert.

PBClientAction

Client action is used to configure the interactions of API sending. The accept types follow the convention of RESTFul which includes:

get/post/put/patch/delete/head

Extension names:

name type note
target String the target client name or alias
name String the action of the API
params Dictionary the parameters for the API
next.done Dictionary the next action triggered by successful response
next.fail Dictionary the next action triggered by error response

PBTriggerAction

Trigger action is used to configure the interactions of events. Accept types:

trigger

Method trigger which used to trigger the method(limit in 5 parameters) of the specified object.

Extension names:

name type note
target String the object, should be an expression refer to Datasource
name String the selector name (as NSStringFromSelector)
next.done Dictionary the next action triggered by the trigger done.

and the params accept:

name type note
arg[0-4] String the argument for the specified index
ret String the return type of the selector (default is void)

Custom Action

You can define your Action with a type unmentioned above.

In following example, we try to implement an Action with a type of toast.

Firstly, we create a class which extends PBAction, named as PBMyCustomAction.

Then implement the run method in PBMyCustomAction.m:

@implementation PBMyCustomAction

@pbaction(@"toast")
- (void)run:(PBActionState *)state {
    UIView *rootView = state.context.supercontroller.view;
    MBProgressHUD *hud = [MBProgressHUD showHUDAddedTo:rootView animated:YES];
    hud.mode = MBProgressHUDModeText;
    hud.bezelView.backgroundColor = [UIColor colorWithWhite:0 alpha:.8f];
    hud.label.textColor = [UIColor whiteColor];
    hud.label.text = self.params[@"message"];
    hud.label.textColor = [UIColor whiteColor];
    [hud hideAnimated:YES afterDelay:1.f];
    dispatch_after(dispatch_time(DISPATCH_TIME_NOW, (int64_t)(1 * NSEC_PER_SEC)), dispatch_get_main_queue(), ^{
        [self dispatchNext:@"done"];
    });
}

@end

Note some details:

  1. Use @pbaction annotation to register an action with type of toast (If got a series types, you can use @pbactions like @pbactions(@"get", @"post")))
  2. Use self.params to get the user-defined parameters
  3. Use [self dispatchNext:@"done"] to trigger the next action configured by next.done

Now we can configure this action by the Plist:

KeyTypeValue
actionDictionary(1 item)
typeStringtoast
paramsDictionary(0 items)
messageStringHello Toast with Pbind Action!

And a next.done to link to next action:

KeyTypeValue
actionDictionary(2 items)
typeStringtoast
paramsDictionary(0 items)
messageStringHello Toast with Pbind Action!
next.doneDictionary(1 item)
typeStringtoast
paramsDictionary(0 items)
messageStringToast Done!

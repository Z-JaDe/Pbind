Create cell layout

Firstly we create a cell layout hero.cell.plist:

The propositional naming convention is: [module].[type].plist, the type here is UITableViewCell and we use cell as abbr.

KeyTypeValue
RootDictionary(2 items)
viewsDictionary(2 items)
nameDictionary(0 items)
clazzStringUILabel
descDictionary(0 items)
clazzStringUILabel
formatsArray(0 items)
Item 0String|-[name]-|
Item 1String|-[desc]-|
Item 2StringV:|-[name]-[desc]-|

Thereinfo:

  • views - The views(dictionary) to be created

    • each key is as the view alias
    • each value is used to describe the view properties

  • formats - The official constraint formats

And a metrics for view margins which is not configured here.

The above three properties are related the official constraint API:

[NSLayoutConstraint constraintsWithVisualFormat:formats[i] options:0 metrics:metrics views:views]

More details of the constraint format language can refer to the official document about VFL.

By the way, Pbind introduce a constraints to describe the complicated constraint formats which are not supported by VFL. You can also get the details in Layout mapper.

Use the layout

Add a row property under the properties node of main.tableview.plist:

KeyTypeValue
rowDictionary(1 item)
layoutStringhero.cell
propertiesDictionary(0 items)
@name.textString.$name
@desc.textString.$desc

row related to PBRowMapper which will map the data for the cell created by PBTableView's cellForRowAtIndexPath.

layout here set to the created hero.cell (note that we exclude the '.plist' suffix here).

properties cell properties which will be map to the created cell.

Here we note that:

  • @name and @desc refer to the view alias in the layout.
  • .$name and .$desc refer to the keyed-value of the related cell data.

Auto-height

Actually we had restrain the two labels fill the cell vertically by V:|-[name]-[desc]-|.

What we needs to do is to configure line wrap for the @desc label. Let's add a numberOfLines property and set to 0 for the desc label in the hero.cell.plist:

KeyTypeValue
propertiesDictionary(0 items)
numberOfLinesString0

The properties is well to known, all the property binding is owe to it.

Save files:

In reality, we may needs to make some little tweaking for the UI. With Plist this would be quite easy.

Tweaking

Cell separator

Note the system related API:

UITableViewCell.separatorInset = UIEdgeInsets(top, left, bottom, right);

Let's make the left of the edge insets to 0 to let the separator fill the bottom of the cell in main.tableview.plist > row > properties:

KeyTypeValue
separatorInsetString{0,0,0,0}

Save:

View margins

Here we configure the metrics mentioned above to control the view margins in hero.cell.plist:

KeyTypeValue
metricsDictionary(0 items)
leftString96

And just use the left metric in the constraint formats:

KeyTypeValue
formatsArray(0 items)
Item 0String|-left-[name]-|
Item 1String|-left-[desc]-|
Item 2StringV:|-[name]-[desc]-|

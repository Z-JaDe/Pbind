Detail table view

Create a hero-detail.tableview.plist:

KeyTypeValue
RootDictionary(1 item)
propertiesDictionary(2 items)
rowDictionary(1 item)
layoutStringhero-detail.cell
rowsArray(2 items)
Item 0Dictionary(1 item)
propertiesDictionary(2 items)
@title.textStringName
@value.textString$name
Item 1Dictionary(1 item)
propertiesDictionary(2 items)
@title.textStringDescription
@value.textString$desc

Thereinto the row assign a base layout as hero-detail.cell and the rows configure the data for each row.

When we both specified row and rows, the row will be took as the base configuration something like the base class.

Router

Back to the main.tableview.plist, let's configure the row actions for each cell:

KeyTypeValue
actionsDictionary(1 item)
selectDictionary(3 items)
typeStringpush
targetStringPBTableViewController
paramsDictionary(3 items)
titleString.$name
plistStringhero-detail.tableview
dataString.$

Action is another base component of Pbind, used to define the interactions. When the action type specified as push, we will push the target page and initialize the page properties by params. The related codes are:

UIViewController *nextController = [[PBTableViewController alloc] init];
nextController.title = self.data.name;
nextController.plist = @"hero-detail.tableview";
nextController.data = self.data;
[self.supercontroller.navigationController pushViewController:nextController animated:YES];

More details of actions can refer to Action component.

We note that the actions here is a Dictionary and use the select key to relate to the select event of each cell(didSelectCellAtIndexPath:), you can also find more action keys in Row mapper.
